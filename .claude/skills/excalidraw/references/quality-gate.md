# Excalidraw Quality Gate

Run this checklist before delivering a diagram.

## Readability

- Text must be readable in the chat preview at normal size.
- Do not put more than 2 short lines inside an ellipse or small card.
- For names, prefer surname or short label inside the visual mark; put full name, club, owner, metric, or description in a side panel.
- Use at least 18 px text for secondary labels and 22 px or larger for primary labels in composed visuals.
- Increase the canvas or simplify the diagram if labels wrap awkwardly.

## Layout Fit

- The chosen layout must match the subject:
  - Tactical roster: pitch + player markers + side panel.
  - Journey/process: columns or swimlanes.
  - Architecture: layered zones.
  - Comparison: side-by-side panels or matrix.
- Avoid forcing specialized visual subjects into a generic row of boxes.
- Leave generous spacing between shapes; avoid touching or overlapping text.

## Information Design

- Separate headline, visual body, and context/footer.
- Make the most important takeaway visible at a glance.
- Use color to encode meaning, not decoration only.
- Do not include every fact inside the primary visual if it hurts comprehension.

## Delivery

- If the diagram uses an Excalidraw Web link, also provide a preview image.
- If the preview is hard to read, revise before responding.
- Mention any important caveat, such as "formation is illustrative, not official."
